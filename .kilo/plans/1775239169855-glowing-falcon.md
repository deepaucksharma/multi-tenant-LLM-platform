## Architecture Analysis

### 1. New Backend Abstraction (`inference/model_backend.py`)

**Design Pattern**: Strategy Pattern + Factory

```
get_model_backend()
     │
     ├── HFBackend (HuggingFace + PEFT)
     │   └── Wraps AdapterManager
     │
     └── OllamaBackend (HTTP to Ollama)
         └── Resolves tenant → model mapping
```

**Key Classes**:

| Class | Responsibility |
|-------|---------------|
| `HFBackend` | Existing local inference with adapter hot-swap |
| `OllamaBackend` | HTTP calls to Ollama, tenant-specific model resolution |
| `get_model_backend()` | Factory with "auto" detection |

**Environment Variable Resolution**:
```python
OLLAMA_MODEL_SIS_SFT  # Tenant+Type specific (highest priority)
OLLAMA_MODEL_SIS      # Tenant specific
OLLAMA_MODEL_SFT      # Type specific  
OLLAMA_MODEL_DEFAULT  # Fallback
OLLAMA_MODEL          # Last resort
```

### 2. App.py Changes

**Refactoring**:
- Replaced direct `AdapterManager` calls with `get_model_backend()` abstraction
- Backend now handles: `prepare_route()`, `generate()`, `generate_stream()`, `get_model_label()`, `get_stats()`

**New Features**:
- `/backend/status` endpoint for debugging backend selection
- Streaming endpoint now includes grounding verification
- Dynamic health status ("healthy" vs "degraded") based on backend availability

### 3. Adapter Hot-Swap Improvements

**Before** (lines 141-155):
- Try/except block that would **destroy model** on failure
- Re-wrapping caused loss of already-loaded adapters
- Concurrent requests could fail when model object was replaced

**After**:
- Only logs failure, preserves active adapter
- Uses `load_adapter()` + `set_adapter()` API to keep multiple adapters in memory
- Pre-check: `if adapter_key not in self._model.peft_config` before loading

### 4. Ollama Integration

**Tenant Model Registration** (`scripts/register_ollama_models.py`):
- Creates Modelfile per tenant/type
- Embeds tenant system prompt
- Uses `ollama create` to build persistent aliases

**Benefits**:
- Stable model names: `tenant-sis-sft`, `tenant-mfg-dpo`
- Pre-configured prompts without runtime injection
- Reuses Ollama's built-in GPU management

### 5. AMD GPU Support

**Script**: `setup_rocm.sh`
- Adds ROCm 6.0 repository
- Installs rocm-hip-sdk, rocm-libs
- Configures environment variables
- Installs PyTorch with ROCm support
- Updates .env device setting

**Documentation**: `AMD_GPU_SETUP.md`
- Troubleshooting for WSL2 AMD GPU
- Performance benchmarks by GPU model
- Memory optimization tips

## Architectural Assessment

### Strengths

1. **Backend Extensibility**: Adding new backends (vLLM, TGI) requires only new class implementing interface
2. **Ollama Auto-Detection**: "auto" mode gracefully falls back to HF if Ollama unavailable
3. **Tenant Model Isolation**: Environment variable resolution allows different models per tenant
4. **Hot-Swap Reliability**: Adapter errors no longer corrupt model state

### Concerns

1. **Ollama Adapter Support**: OllamaBackend doesn't support PEFT adapters — Modelfile approach is workaround
2. **Duplicate Code**: `get_model_label()` defined twice in HFBackend (lines 67-68 and 75-76)
3. **Streaming Grounding**: Calculated post-generation, adds latency; could be done during generation
4. **Backend Singleton**: Global `_backend` doesn't reset between tests

## Summary

This change introduces a **production-ready dual-backend inference architecture** that enables:
- **Local development**: HuggingFace with PEFT adapters
- **AMD GPU deployment**: Ollama with tenant-specific models
- **Graceful degradation**: Auto-detection when backends unavailable

The refactoring maintains backward compatibility while adding significant operational flexibility.
